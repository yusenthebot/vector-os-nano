# v2.3 Live REPL Smoke Checklist — Go2 Perception Pipeline

Manual verification checklist for v2.3 end-to-end in `vector-cli` REPL,
against MuJoCo sim with `with_arm=1`.

Target: 2026-04-20 onward, after v2.3 commits merged. The new
behaviour: `抓 X` now works against an **empty world_model** —
perception auto-populates it via Qwen VLM + depth projection.

---

## Prerequisites

- `.venv-nano` activated (or `vector-cli` entry point on PATH)
- `OPENROUTER_API_KEY` set (or in `config/user.yaml` under `llm.api_key`).
  Value must be an OpenRouter key with access to
  `qwen/qwen2.5-vl-72b-instruct`.
- No stale `go2_vnav_bridge` subprocesses:
  ```bash
  pgrep -af go2_vnav | grep -v grep
  # should be empty
  ```
- Terminal ready.

## Test Matrix

| # | Command | Expected Outcome | Failure Diagnosis |
|---|---------|------------------|-------------------|
| 1 | `go2sim with_arm=1` | Sim launches: bridge + nav stack + rviz up in <30s. Log contains the line `[sim_tool] Go2 perception + calibration wired (Qwen)`. `/tools` shows `pick_top_down`, `place_top_down`, `mobile_pick`, `mobile_place`, `detect_objects`. `agent._perception` is a `Go2Perception`; `agent._calibration` is a `Go2Calibration`. World model starts EMPTY (this is correct in v2.3). | **"wired" line missing** → wire-up failed; check `OPENROUTER_API_KEY` is visible to the shell AND `config/user.yaml`. Try `echo $OPENROUTER_API_KEY`. Also check `logger.warning("[sim_tool] Perception wire-up failed: ...")` in log. |
| 2 | `检测物体` or `detect_objects query="all objects"` | `DetectSkill` runs VLM → reports `count=N` (0-5 typical depending on scene), adds N `ObjectState` to world_model. Each logged with `camera(x,y,z)` → `base(x,y,z)` transform line. Qwen call latency 1-4 s on first call. | **`no_perception`** diagnosis → `agent._perception` is None; re-check step 1. **Timeout / RuntimeError from VLM** → OpenRouter network issue or quota. Re-check key + run `curl -s https://openrouter.ai/api/v1/auth/key -H "Authorization: Bearer $OPENROUTER_API_KEY"`. |
| 3 | `抓起蓝色瓶子` (pick up blue bottle) — with world_model still empty | `MobilePickSkill` triggered. First `_resolve_target` misses. Auto-detect kicks in: logs `[MOBILE-PICK] world_model miss; auto-detect query='blue bottle'`. `DetectSkill` runs, populates world_model with a "blue bottle" ObjectState. Resolve retries → hit. Navigate + wait_stable + pick. Final result `held=True`, total elapsed ≤30 s. | **`object_not_found` despite perception wired** → `context.calibration` may be None (check log during wire-up). **Auto-detect runs but returns count=0** → VLM didn't see the bottle. Try narrower query: `抓瓶子` or `detect_objects query="bottle"` to confirm VLM sees something. **Retry resolves but picks wrong object** → VLM returned a bbox over a different object; re-plan or `抓红色瓶子` if there are multiple. |
| 4 | Re-run step 3 immediately: `抓起蓝色瓶子` | Second call finds the object in world_model (just populated by step 3). No auto-detect this time; goes straight to resolve → pick. Faster (no VLM call; ≤15 s). | **Auto-detect re-runs** → world_model not persisted between invocations (unexpected — `ObjectState` should remain). Check `scene_graph.yaml` was not cleared. |
| 5 | Simulate VLM failure: unset `OPENROUTER_API_KEY` before launch, repeat step 1 then 3 | Step 1: wire-up block logs `Perception wire-up failed: ...`. `agent._perception = None`. Step 3: MobilePickSkill returns `object_not_found` with `Known pickable objects: []` — **no crash**, clean error, identical to v2.2 behavior. | Any AttributeError / crash here is a regression — the graceful-degradation path (v2.3 AC-7) is broken. |

---

## Success Criteria

- All 5 rows execute as expected.
- The `[sim_tool] Go2 perception + calibration wired (Qwen)` log line
  appears in step 1 (when key is configured).
- Step 3 completes end-to-end with held=True on an empty world_model.
- Step 5 degrades gracefully with no crash when perception is
  unavailable.
- No `RuntimeError: Executor is already spinning` (v2.2 regression
  check).
- No `No perception backend available` for MobilePick on step 3 when
  perception is wired (v2.3 resolves that path).

## What Changed in v2.3

### New: QwenVLMDetector (grounded 2D bbox)

`vector_os_nano/perception/vlm_qwen.py`. Calls OpenRouter
`qwen/qwen2.5-vl-72b-instruct` with a JSON-schema prompt. Returns
`list[Detection]` with pixel-space bboxes. Reuses v2.2's JPEG /
retry infrastructure; honours `VECTOR_VLM_URL` (local endpoint) and
`VECTOR_VLM_MODEL` env overrides.

### New: Go2Perception (PerceptionProtocol for Go2)

`vector_os_nano/perception/go2_perception.py`. Composes
`Go2ROS2Proxy` camera frames + `QwenVLMDetector` into the
`PerceptionProtocol` surface that `DetectSkill` already expects.
`track()` is single-shot: bbox centre + median bbox depth with IQR
outlier reject → `Pose3D` in camera frame. No EdgeTAM, no masks
(deferred to v2.4).

### New: Go2Calibration (camera→world)

`vector_os_nano/perception/go2_calibration.py`. Pose-driven
calibration — reads `Go2ROS2Proxy.get_camera_pose()` per call and
transforms OpenCV-frame (right, down, forward) camera points to
world xyz using the MuJoCo xmat convention. No paired-point
fitting.

### Wired: sim_tool `_start_go2(with_arm=True)`

After Piper proxies connect, constructs
`Go2Perception(camera=base, vlm=QwenVLMDetector(api_key=...))` and
`Go2Calibration(base_proxy=base)`, assigns to `agent._perception` /
`agent._calibration`. Failure is logged and degraded gracefully
(both set to None). `agent._vlm` (Go2VLMPerception, for
describe_scene/identify_room) coexists unchanged.

### Changed: MobilePickSkill auto-detect retry

`vector_os_nano/skills/mobile_pick.py`. On first `_resolve_target`
miss AND `context.perception` + `context.calibration` both
available, lazily invokes `DetectSkill` with a CN→EN query
translated via `label_to_en_query`, then retries resolve. Failure
path unchanged (graceful `object_not_found` with known_objects
list).

### New: `label_to_en_query` helper

`vector_os_nano/skills/utils/__init__.py`. Strips "的", reuses
`_normalise_color_keyword`, applies a small CN noun map (瓶子 →
bottle, 杯子 → cup, etc). Returns `None` for empty/None input.

### Not changed

- Any ROS2 topic / service / action / message interface.
- Any Python dependency.
- Any MJCF, launch script, nav stack, MPC, or arm control.
- Any existing test (17 existing mobile_pick tests preserved).
