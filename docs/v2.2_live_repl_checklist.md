## What Changed in v2.2

### Bug 1 Fix: Executor Spinning Error

**Problem**: Three proxies (Go2ROS2Proxy, PiperROS2Proxy, PiperGripperROS2Proxy) each called `rclpy.spin()` in separate threads. Only one process-global executor can spin; the 2nd and 3rd raised `RuntimeError: Executor is already spinning`.

**Solution**: New `Ros2Runtime` singleton maintains ONE `MultiThreadedExecutor`. Each proxy calls `executor.add_node(n)` instead of starting its own spin thread. Single background thread spins the shared executor.

**Files modified**: `hardware/sim/go2_ros2_proxy.py`, `hardware/sim/piper_ros2_proxy.py`, `hardware/sim/piper_ros2_proxy.py` (gripper), `vcli/tools/sim_tool.py`.

### Bug 2 Fix: Chinese Color Keywords Not Recognized

**Problem**: User command "抓绿色" (grab green) resolved to `object_label="绿色"`. WorldModel search uses substring match on stored labels ("green bottle"). Chinese color characters don't substring-match English colors.

**Solution**: New `_normalise_color_keyword()` helper in PickTopDownSkill maps Chinese color words ("绿/绿色" → "green", "红/红色" → "red", "蓝/蓝色" → "blue", etc.) before world_model lookup. Falls back to single-candidate if label is unknown.

**Files modified**: `skills/pick_top_down.py` (PickTopDownSkill._resolve_target).

### Bug 3 Fix: Unnecessary Perception Fallback

**Problem**: When PickTopDownSkill couldn't resolve a label, VGG tried to decompose with a `detect_*` fallback. But `context.perception` is not wired in `with_arm` mode — only `context.vlm` (vision-language model) is available. This logged "No perception backend" warning and failed the goal.

**Solution**: VGG `GoalDecomposer` now respects the skill's parameter `source` metadata. If a skill declares `source: world_model.*` or `source: static`, skip the perception fallback and return "Cannot resolve" to the user instead. Clearer UX; aligns with the skill's contract ("objects are known a priori").

**Files modified**: `vcli/cognitive/goal_decomposer.py` (GoalDecomposer system prompt builder).

### New Skills

- **PlaceTopDownSkill** (`skills/place_top_down.py`): Mirror of PickTopDownSkill. Pre-hover above target → descend → open gripper → lift (not home). Explicit or receptacle-based target resolution.

- **MobilePickSkill** (`skills/mobile_pick.py`): Composition: resolve target → compute approach pose → navigate (if needed) → wait_stable → delegate to pick_top_down. Aliases: "去拿", "去抓", "拿来", "取来", "去取", "fetch", "go grab", "go get".

- **MobilePlaceSkill** (`skills/mobile_place.py`): Composition: resolve target → compute approach pose → navigate (if needed) → wait_stable → delegate to place_top_down. Mirrors mobile_pick pattern.

### Utilities

- **compute_approach_pose()** (`skills/utils/approach_pose.py`): Compute the (x, y, yaw) approach position for the dog to stop at a safe distance (default 0.55 m) from a target, facing it.
